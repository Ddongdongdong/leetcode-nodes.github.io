解法1：
合并k个有序链表的时候，我首先想到的是用类似于二分法的方法去把整个数组分割一下再排序：
代码实现：
class Solution:
    def mergeKLists(self, lists):
        if not lists:
            return None

        interval = 1
        n = len(lists)

        while interval < n:
            for i in range(0, n - interval, interval * 2):
                lists[i] = self.mergeTwoLists_iter(lists[i], lists[i + interval])
            interval *= 2

        return lists[0]

    def mergeTwoLists_iter(self, l1, l2):
        dummy = ListNode(-1)
        p = dummy
        while l1 and l2:
            if l1.val <= l2.val:
                p.next = l1
                l1 = l1.next
            else:
                p.next = l2
                l2 = l2.next
            p = p.next
        p.next = l1 if l1 else l2
        return dummy.next
这样就可以把 K 路问题拆成很多次类似于21题的那种两个有序链表的合并问题。
解法2：
按照最小根堆的方法将链表中头节点挨个取出，用到优先级队列这种数据结构，把链表节点放入一个最小堆，就可以每次获得 k 个节点中的最小节点。方法很巧妙，思路也很好，比第一种方法的逻辑性更强。

代码实现:
import heapq

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

    # 重载比较运算符，方便将 ListNode 加入最小堆
    def __lt__(self, other):
        return self.val < other.val

class Solution:
    def mergeKLists(self, lists):
        if not lists:
            return None
        # 虚拟头结点
        dummy = ListNode(-1)
        p = dummy
        # 优先级队列，最小堆
        pq = []
        # 将 k 个链表的头结点加入最小堆
        for i, head in enumerate(lists):
            if head is not None:
                heapq.heappush(pq, (head.val, i, head))

        while pq:
            # 获取最小节点，接到结果链表中
            val, i, node = heapq.heappop(pq)
            p.next = node
            if node.next is not None:
                heapq.heappush(pq, (node.next.val, i, node.next))
            # p 指针不断前进
            p = p.next
            
        return dummy.next

每次从 K 个链表中找最小的头结点接到结果链表中,使用最小堆可以高效地合并 K 个有序链表。
